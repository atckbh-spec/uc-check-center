# UC Check 롤백 및 장애 대응 Runbook

목적: 현장 운영 중 출석 오류, 키오스크 장애, 배포 장애가 발생했을 때 스태프가 바로 대응할 수 있도록 한다.

---

## 1. 장애 등급

| 등급 | 상황 | 대응 |
|---|---|---|
| SEV-1 | 출석 체크/잔여횟수 차감이 전체 실패 | 즉시 키오스크 중단, 수기 출석 기록, 롤백 검토 |
| SEV-2 | 일부 회원 출석 오류, 중복 출석, 취소 실패 | 관리자 확인 후 수동 조정, audit 기록 |
| SEV-3 | 대시보드/리포트 표시 오류 | 운영 계속, 개발 수정 요청 |
| SEV-4 | UI 문구/디자인 문제 | 다음 배포에 반영 |

---

## 2. 현장 임시 운영 절차

앱이 멈추거나 출석 체크가 실패하면 즉시 다음 방식으로 전환한다.

1. 태블릿 키오스크 화면을 닫는다.
2. 프론트/코치가 수기 출석표를 연다.
3. 회원명, 시간, 회원권, 담당자, 특이사항을 기록한다.
4. 앱 복구 후 admin이 출석을 일괄 입력한다.
5. 잔여횟수 오류가 있으면 사유와 함께 조정한다.

수기 출석표 필수 항목:

```text
날짜
시간
회원명
전화번호 끝 4자리
회원권명
담당자
출석/노쇼/취소
메모
```

---

## 3. 배포 롤백 절차

Vercel 기준:

1. Vercel Dashboard 접속
2. UC Check 프로젝트 선택
3. Deployments 탭 이동
4. 직전 정상 배포 선택
5. Promote to Production 실행
6. `/login`, `/dashboard`, `/kiosk` 정상 확인
7. 장애 원인 기록

롤백 후 확인:

- [ ] `/login` 접근 가능
- [ ] `/dashboard` 로그인 후 접근 가능
- [ ] `/kiosk` 표시 가능
- [ ] 출석 체크 테스트 가능
- [ ] Supabase 데이터가 손상되지 않았음

---

## 4. 데이터 오류 복구

### 4.1 잘못된 출석 체크

처리:

1. `/attendance/today` 또는 회원 상세에서 해당 기록 확인
2. admin/owner가 출석 취소
3. 취소 사유 입력
4. 잔여횟수 복구 확인
5. audit_logs 확인

사유 예시:

```text
키오스크 회원 선택 오류로 인한 출석 취소
스태프 오입력으로 인한 잔여횟수 복구
중복 체크 확인으로 인한 출석 취소
```

### 4.2 잔여횟수 오류

처리:

1. 회원 상세 이동
2. 회원권 카드에서 잔여횟수 조정
3. `+1`, `-1` 선택
4. 사유 입력
5. 저장 후 audit 확인

사유 예시:

```text
수기 출석표 기준 복구
지난 출석 취소 누락 보정
회원권 등록 시 초기 횟수 오입력 보정
```

### 4.3 노쇼 오기록

MVP 정책상 노쇼는 차감하지 않는다. 잘못 기록된 노쇼는 audit 보존을 위해 삭제하지 않고, 메모로 정정 기록을 남긴다.

권장 처리:

1. 회원 메모에 정정 기록 추가
2. 필요하면 admin이 DB에서 상태 변경, 변경 사유 audit 기록
3. 다음 버전에서 no_show 취소 UI 추가 검토

## 5. Supabase 장애 대응

증상:

```text
로그인 실패
출석 체크 실패
대시보드 데이터 로딩 실패
키오스크 후보 조회 실패
```

확인 순서:

1. Supabase 프로젝트 상태 확인
2. 환경변수 누락 여부 확인
3. service role key 만료/오류 확인
4. RLS policy 최근 변경 확인
5. RPC `check_in_member` 실행 권한 확인

임시 대응:

- 키오스크 운영 중지
- 수기 출석표 전환
- 복구 후 일괄 입력

## 6. 개인정보 사고 대응

예시:

```text
키오스크에 전체 전화번호가 표시됨
회원 메모가 키오스크에 표시됨
비로그인 상태에서 관리자 화면이 보임
```

즉시 조치:

1. 해당 화면 접근 차단
2. Vercel 직전 정상 버전으로 롤백
3. Supabase anon/service key 노출 여부 확인
4. 로그 확인
5. 영향 범위 기록
6. 재발 방지 패치 후 재배포

기록 항목:

```text
발견 시간
발견자
노출 화면
노출 정보 유형
영향 회원 수
즉시 조치
재발 방지책
```

## 7. 배포 후 24시간 모니터링

배포 후 첫날은 아래 항목을 확인한다.

- [ ] 출석 성공 수
- [ ] 출석 실패 수
- [ ] PIN 실패 수
- [ ] 후보 없음 수
- [ ] 후보 과다 수
- [ ] 출석 취소 수
- [ ] 잔여횟수 조정 수
- [ ] 노쇼 기록 수
- [ ] 스태프 문의 건수

이상 신호:

```text
PIN 실패 급증
출석 취소 급증
잔여횟수 조정 급증
후보 없음 급증
키오스크 출석보다 staff 출석이 과도하게 많음
```
