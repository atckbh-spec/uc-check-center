# UC Check 현장 QA 체크리스트

목적: Urban Conditioning 현장에서 태블릿, 스태프 PC, 실제 운영 플로우 기준으로 배포 전 문제를 찾는다.

권장 환경:
- 태블릿 1대: `/kiosk` 전용
- 스태프 PC/노트북 1대: `/dashboard`, `/members`, `/attendance/today`
- 테스트 계정 3개: owner, admin, coach
- 테스트 회원 8명 이상: 정상, 재등록 대상, 장기 미방문, 잔여 0회, 동일 전화번호 뒷자리 후보 포함

---

## 0. QA 시작 전 차단 조건

아래 항목 중 하나라도 실패하면 현장 QA를 시작하지 않는다.

- [ ] `pnpm install` 성공
- [ ] `pnpm typecheck` 성공
- [ ] `pnpm lint` 성공
- [ ] `pnpm build` 성공
- [ ] `pnpm qa:build-output` 성공
- [ ] Supabase migration 적용 완료
- [ ] owner 스태프 계정 로그인 가능
- [ ] `/dashboard` 비로그인 접근 시 `/login`으로 이동
- [ ] `/settings/staff` 비로그인 접근 시 `/login`으로 이동
- [ ] 키오스크에서 전체 전화번호, 메모, 관리자 정보가 노출되지 않음
