# UC Check 스태프 교육 가이드

교육 목표: 스태프가 10분 안에 UC Check의 핵심 운영 흐름을 익히고, 현장에서 출석 오류를 복구할 수 있게 한다.

---

## 1. 스태프가 매일 할 일

1. 로그인 후 `/dashboard` 확인
2. 오늘 출석 수 확인
3. 재등록 대상 확인
4. 장기 미방문 회원 확인
5. 키오스크가 켜져 있는지 확인
6. 출석 오류가 있으면 당일 바로 수정

---

## 2. 회원 셀프 체크인 안내 문구

회원에게 이렇게 안내한다.

```text
입구 태블릿에서 휴대폰 번호 끝 4자리를 입력해 주세요.
본인 이름을 확인하고 출석 체크를 누르면 됩니다.
동명이인이나 같은 번호 후보가 있으면 스태프에게 말씀해 주세요.
```

개인 PIN을 쓰는 운영이라면:

```text
본인 확인을 위해 개인 출석 PIN을 한 번 더 입력해 주세요.
PIN을 모르시면 스태프에게 말씀해 주세요.
```

---

## 3. 출석 오류 처리

### 잘못 출석 체크된 경우

1. `/attendance/today` 이동
2. 해당 출석 기록 찾기
3. 출석 취소 클릭
4. 사유 입력
5. 잔여횟수 복구 확인

### 잔여횟수가 틀린 경우

1. 회원 상세 이동
2. 회원권 카드 확인
3. admin/owner가 잔여횟수 조정
4. 사유 입력
5. 저장

---

## 4. 재등록 대상 처리

대상 기준:

```text
잔여 1~3회
```

처리:

1. `/dashboard` 재등록 대상 확인
2. 회원 상세 이동
3. 최근 출석 패턴 확인
4. 상담 또는 재등록 안내
5. 메모에 상담 내용 기록

메모 예시:

```text
재등록 안내 완료. 다음 방문 시 20회권 안내 예정.
```

---

## 5. 장기 미방문 처리

대상 기준:

```text
최근 방문일 14일 이상 전
```

처리:

1. `/dashboard` 장기 미방문 대상 확인
2. 회원 상세에서 출석 패턴 확인
3. 연락 여부 결정
4. 메모 기록

메모 예시:

```text
2주 이상 미방문. 컨디션 확인 연락 필요.
```

---

## 6. 권한 기준

| 기능 | 가능 역할 |
|---|---|
| 출석 체크 | owner, admin, coach, front_desk |
| 노쇼 기록 | owner, admin, coach, front_desk |
| 회원 등록 | owner, admin, front_desk |
| 출석 취소 | owner, admin |
| 잔여횟수 조정 | owner, admin |
| 직원 관리 | owner |

---

## 7. 절대 하면 안 되는 것

- 회원 메모를 키오스크 화면에 보여주지 않는다.
- 회원 전체 전화번호를 입구 태블릿에서 보여주지 않는다.
- 잔여횟수를 사유 없이 조정하지 않는다.
- 출석 오류를 다음 날로 미루지 않는다.
- 실회원 데이터로 테스트하지 않는다.
