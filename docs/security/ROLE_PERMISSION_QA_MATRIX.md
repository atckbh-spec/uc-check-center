## 역할 정의

| 역할 | 목적 |
|---|---|
| Owner | 전체 운영, 직원 관리, 보안/감사 확인 |
| Admin | 회원/회원권/출석 운영 관리 |

| Front Desk | 회원 등록, 출석 체크, 기본 접수 운영 |

## 수동 테스트 절차

각 역할 계정으로 로그인한 뒤 아래를 확인한다.

1. 허용되지 않은 버튼이 화면에서 숨겨져 있는지 확인한다.
2. URL 직접 입력으로 접근했을 때도 차단되는지 확인한다.
3. 브라우저 개발자도구 또는 직접 form submit으로 서버 액션을 호출해도 차단되는지 확인한다.
4. Supabase RLS가 앱 권한과 동일하게 제한하는지 `pnpm security:db`로 확인한다.

## 실패 시 수정 기준

- 화면에서 버튼만 숨기는 것은 충분하지 않다.
- Server Action 내부에서 `requireRole()`을 다시 확인해야 한다.
- Supabase RLS도 동일 역할 기준으로 제한해야 한다.
- 출석/잔여횟수 변경은 직접 table update가 아니라 RPC 중심이어야 한다.
